# Acceptance Gates

## Correctness
- [ ] Exhaustive truth-table tests.
- [ ] Distinct disposable base/head worlds use identical probe definitions.
- [ ] Setup errors/timeouts/nondeterminism become inconclusive.
- [ ] Capsules validate, bind exact SHAs, and replay verifies hashes.

## Adversarial strength
- [ ] At least five changed-TypeScript AST operators.
- [ ] Invalid mutants excluded, not counted killed.
- [ ] Survivors labeled evidence gaps, not bugs.
- [ ] Authorization fixture proves weak-vs-strong evidence.

## Safety
- [ ] Workspace path escape blocked; commands timeout/resource bounded.
- [ ] Common secrets redacted; no telemetry/upload by default.
- [ ] Fork guidance forbids secrets/privileged execution.

## Product/release
- [ ] Clean install + demo under five minutes on documented environment.
- [ ] Terminal/HTML agree; HTML accessible and self-contained.
- [ ] Least-privilege GitHub Action; three reproducible fixtures.
- [ ] Tests/typecheck/package check pass; docs match behavior.
- [ ] No critical TODO/FIXME; limitations prominent; `BUILD_REPORT.md` complete.

## Strategic differentiation gates
- [ ] Capsule can be verified without originating model/SaaS.
- [ ] Maintainer policy is machine-validated and enforced on fixtures.
- [ ] At least one agent integration emits claims and attaches a capsule.
- [ ] Plugin conformance fixture validates one external-style adapter.
- [ ] Benchmark methodology includes hidden siblings, checker isolation and disclosure policy.
- [ ] Public messaging never claims formal proof/certification.
- [ ] Demo visibly distinguishes the project from reviewer/test-generation incumbents.
