# GTM and OpenClaw-style Traction Loops

## Launch hook
**The code passed. The evidence failed.** Demo a green AI PR whose new test is vacuous, whose authorization invariant regresses, and whose auth-removal mutant survives.

## Compounding loops
1. **Capsule loop:** capsule in PR → reviewer sees it → maintainer adopts policy → future contributors install verifier.
2. **Agent loop:** skills/plugins make agents attach capsules → examples spread across model communities.
3. **Benchmark loop:** verifiability rankings drive model/vendor sharing and competition.
4. **Adapter loop:** framework contributors add adapters → new communities arrive.
5. **Badge loop:** repositories publish “Evidence Required/Verified” badges linked to live methodology.

## 14 days
Ship 3 fixtures; test with 10 maintainers; publish capsule spec; add one agent skill; run against 10 permissive PRs with disclosure; Show HN only after every claim is replayable.

## 30 days
TypeScript stable; Python adapter alpha; GitHub Action; policy bot; benchmark v0; 5 agent integrations; 25 public capsules; weekly “Evidence Failed” teardown.

## North stars
Third-party replays, repositories requiring capsules, agent integrations, adapter contributors, capsules generated, regressions/vacuity found, repeat weekly users—not stars alone.
